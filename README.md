## Overview
This project investigates the relationship between structural job quality (autonomy, safety, and management support) and mental well-being (WHO-5 index) among South Korean wage workers, using the **7th Korea Working Conditions Survey (KWCS) 2023**. 

## Project Structure
The repository is organized to ensure full reproducibility of the data processing, statistical analysis, and report generation:

- `report.pdf` / `report.html`: The final academic reports containing all interpretations and visualizations.
- `report.Rmd`: The R Markdown source code used to generate the final reports.
- `README.md`: This file.

### Directories
- **`scripts/`**: Contains the core R scripts for the pipeline.
  - `01_import_data.R`: Downloads and imports the raw KWCS data.
  - `02_cleaning.R`: Prepares the analytic sample, handles missing values, scales the job quality indices, and applies survey weights.
  - `03_analysis.R`: Generates the descriptive statistics, regressions, and outputs the visualizations.
- **`data/`**: Contains the raw (`kwcs_raw.rds`) and processed (`kwcs_cleaned.rds`) datasets.
- **`plots/`**: Contains all PNG visualizations generated by the analysis script.
- **`theme/`**: Contains custom R themes (`theme.R`), CSS (`report.css`), and HTML styling components to format the reports.

## Reproducibility Guide
To fully reproduce the analysis and the final report from scratch, run the scripts in the following order:

1. **Setup Environment**: Ensure you have R installed along with the `tidyverse`, `survey`, `broom`, `bookdown`, and `plotly` packages.
2. **Run Pipeline**:
   - Execute `scripts/01_import_data.R`
   - Execute `scripts/02_cleaning.R`
   - Execute `scripts/03_analysis.R`
3. **Render Report**:
   - Open `report.Rmd` in Positron or RStudio and click **Knit** (to PDF or HTML).
   - Alternatively, run `rmarkdown::render("report.Rmd", output_format = "all")` in the R console.

## Notes
All statistical models utilize the `survey` package to incorporate the KWCS sampling weights, ensuring nationally representative estimates. Missing values (e.g., "Refused", "Don't Know") have been properly recoded to `NA` prior to analysis.
